делаю т.з. в первый раз, поэтому не на английском языке

--Main:
Создать веб-приложение новостей в кратком и беспристрастном изложении со ссылками на источники.

--Server
OS: Ubuntu 22.04.2 LTS
Web server: Apache/2.4.52
Programming language: PHP 8.1.2-1
Backend framework: Laravel 10.13.0
Backend framework libraries: UI, doctrine/dbal
Frontend framework: отсутствует

--Database:
СУБД: MySQL 8.0.33-0
Имя базы данных: brief_news
Пользователь: brief_news_user
Пароль: 'стандартный' (не отображен, т.к. т.з. будет доступено на GitHub)
Дополнительно: Soft delete для пользователей, статей и ссылок; создать первого root (admin привилегии) пользователя


--Роли и статусы пользователей, разграничение прав:
root: first Admin, не подлежит смене роли, не подлежит смене статуса, не подлежит удалению;
Admin: CRUD учетных записей (так-же смена роли пользователей Admin/Writer/Reader, смена статуса пользователей active/ban, восстановление удаленных пользователей), CRUD статей (так-же восстановление удаленных статей), интерфейс Reader, не подлежит смене статуса;
Writer: RU своей учетной записи (не имеет права смены роли и статуса), CRU своих статей, интерфейс Reader;
Reader: RU своей учетной записи, выбирать категории статей по умолчанию (в Мой профиль), сортировка статей по категориям (на Главной), читать статьи;
Unregistered/Unauthenticated: создание учетной записи Reader роли, сортировка статей по категориям (на Главной), читать статьи;

ban статус: не может создавать и редактировать статьи, оставлять коментарии (при реализации их на сайте);

--Interface:

-Общий стиль - web приложение в серых тонах, подчеркивающий беспристрастность и не отвлекающий от новостей;

-Header - узкая горизонтальная полоса в верхней части экрана, присутствует на всех страницах:
Название сайта - в вверхней левой части Header, большими черными буквами (HTML+CSS).
Строка приветствия - в средней правой части Header, имя и роль пользователя.
Кнопки пользовательского интерфейса - в нижней части Header, с права на лево, для разных ролей разные:
Admin: Выход, Удаленные профили, Мой профиль, Создать профиль, Все профили, Удаленные статьи, Мои статьи, Создать статью, Главная;
Writer: Выход, Мой профиль, Мои статьи, Создать статью, Главная;
Reader: Выход, Мой профиль, Главная;
Unregistered/Unauthenticated: Регистрация, Вход, Главная.

-Footer: Отсутствует на всех страницах :)

-Страница Главная:
Панель сортировки статей по рубрикам и локали:
Горизонтальная полоса из кнопок: Все, Политика, Экономика, Наука, Технологии, Спорт, Остальные, Мировые, Локальные;
Если нажата "Все" - остальные отжаты;
Если нажаты или отжаты все кнопки кроме "Все" - "Все" переходит в нажатое состояние, остальные в отжатое;
"Все" отжата при любом сочетании остальных кнопок;
При выборе одной или нескольких рубрик, локали (Мировые и Локальные) по умолчанию нажаты.

Новости:
Вертикальная лента горизонтальных прямоугольников - заголовков новостей с датой и временем публикации; по 15 на странице.
Сортируются по последней дате. Отображаются статьи выбранных рубрик и локалей.
При нажатии на заголовок раскрывается тело статьи, в конце которой ссылки на источники, псевдоним автора и кнопка свернуть.
При повторном нажатии на заголовок статьи тело статьи скрывается.
У автора статьи (Writer) под заголовком его статей отображается кнопка Редактировать.
У Admin: псевдоним автора - ссылка, по которой можно перейти на профиль автора; под каждой статьей отображаются кнопки Редактировать, Удалить/Восстановить.

Пагинатор:
Выполнен в стиле сайта. Номера страниц со стрелками следующая/предыдущая.

-Страница Мои статьи (Admin/Writer):
Идентична Главной, отображаются статьи только текущего пользователя.

-Страница Создать и Редактировать статью (Admin/Writer):
Рубрики и локаль статьи:
Горизонтально расположеные названия с галочками: Политика, Экономика, Наука, Технологии, Спорт, Остальные, Мировые, Локальные. Изначально все галочки пусты, валидатор требует поставить хотя бы одну галочку в рубриках и хотя бы одну галочку в локали.

Статья:
HTML form с вертикально расположенными полями:
	header
	body
	sources links

В нижней части страницы кнопки Сохранить (Главная), Отмена (Главная).

-Страница Удаленные статьи (Admin):
Идентична Главной, отображаются все удаленные статьи, вместо Редактировать и Удалить кнопка Восстановить статью.

-Страница Регистрации (Unregistered/Unauthenticated):
Стандартная страница Laravel/UI библиотеки оформленная в стиле web приложения.
При регистрации обязательные для заполнения поля:
	name (требования библиотеки)
	email
	password
При регистрации назначаемые по умолчанию поля:
	surname: NULL
	nickname: incognito
	role: reader
	status: active
	rubrick_id: 1
кнопки: Зарегистроваться (Главная), Отмена (Главная).

-Страница Все профили (Admin):
Таблица с полями: ID, Имя, Фамилия, Псевдоним, Роль, Статус, Email, Дата создания, Редактировать, Удалить.
Сортируются по последнему id, по 15 на странице. Для большего удобства чтения имеет разное оформление четных и нечетных строк.

-Страница Создать профиль (Admin):
HTML form с вертикально расположенными полями:
	Выбор Рубрик и локалей по умолчанию
	role
	status
	name
	surname
	nickname
	email
	password
В нижней части страницы кнопки Сохранить изменения (Главная), Отмена (Главная).

-Страница Мой профиль (Admin/Writer/Reader):
В верхней части страницы кастомизатор Рубрик отображаемых по умолчанию пользователю на Главной. Горизонтально расположеные названия рубрик с галочками (для наглядного отличия от рубрик на Главной странице). Сочетание галочек подчиняются тем же правилам, что и кнопки рубрик на Главной странице. Изначально пользователю отображаются все рубрики (All, rubrick_id: 1). Валидатор требует поставить хотя бы одну галочку в рубриках и хотя бы одну галочку в локали.

В средней части страницы HTML form с вертикально расположенными предзаполненными полями:
	ID учетной записи (без возможности изменить)
	email
	name
	surname
	nickname
	Кнопка Изменить пароль (ведущая на отдельную страницу)
В нижней части страницы кнопки Сохранить (Главная), Отмена (Главная).

-Страница Изменить пароль (Admin/Writer/Reader):
Вертикально расположенные поля:
	Введите текущий пароль
	Введите новый пароль
	Введите новый пароль еще раз
В нижней части страницы кнопки Сохранить новый пароль (Мой профиль), Отмена (Мой профиль).

-Страница Редактировать профиль пользователя Name (Admin):
HTML form с вертикально расположенными предзаполненными полями:
	кастомизатор Рубрик и локалей
	id
	created_at
	updated_at
	role
	status
	name
	surname
	nickname
	email
	галочка "Изменить пароль", установить для изменения пароля и тогда поле password должно быть заполнено
	password - пустое поле с фоновой надписью "Will not change (не изменится)"
В нижней части страницы кнопки Сохранить изменения (Все профили), Отмена (Все профили).

-Страница Удаленные профили (Admin):
Идентична Все профили, отображаются все удаленные профили, вместо Редактировать и Удалить кнопка Восстановить профиль.

-Страница Подтверждения удаления:
При удалении статьи или пользователя происходит переход на страицу подтверждения с краткой информацией и кнопками Удалить и Отмена.
